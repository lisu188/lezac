# End-Flow Recovery Subagents - 2026-04-20

This pass used five read-only subagents and integrated their findings into the
local C++ port.

## A - Ghidra/Disassembly Mapper

Ghidra was not available on `PATH`, so the pass used the MZ header mapping
already confirmed for `LEZAC.EXE`:

```text
file_offset = 0x770 + CS_offset
```

Recovered post-game anchors:

- `1000:1b14..1d42` is the post-game dispatcher.
- Mode `1` draws `game over`.
- Mode `2` draws `eccellente>>>` and `hai completato il gioco`, then sets a
  completed-game flag at `DS:208c`.
- Player 1 score is read from `DS:785a`; player 2 score is read from
  `DS:7888`.
- Record qualification compares each score against the current seventh record
  slot immediately before calling `1000:1845`, so player 2 is checked against
  the table after any player 1 insertion.

## B - Gameplay Fidelity

Highest-impact gap: the port had a single score and pending record slot. This
could not represent the original two-player loop.

Integrated:

- Added per-player score storage.
- Attributed objective collection, bonus collection, and bomb-object points to
  the collecting or bomb-owning player.
- Added an end-run record candidate queue that preserves original player order
  and re-checks the current table before each prompt.
- Final-level completion now enters completed-game flow instead of wrapping
  directly back to level 1.
